# Non-Relational Database Documentation

## Overview

This MongoDB schema models physical infrastructure and operational metrics of Brazil's energy distribution networks. The design supports two primary objectives:

1. **Infrastructure Dimensioning:** Identify network capacity and geographic distribution patterns to assess market potential for Tecsys technology (sensing and telemetry).
2. **Loss Prediction:** Geographically locate energy losses and continuity failures to guide infrastructure improvements and predict where Tecsys deployment would have the highest impact.

The data-driven approach enables regulatory compliance (ANEEL reporting) while creating actionable insights for reducing non-technical losses (theft, fraud) and improving service continuity.

## Technology Stack

**Database:** `tecsys` (MongoDB 6.0)  
**Driver:** PyMongo  
**Connection:** Configured via environment variables using `python-dotenv`

## Setup Overview

`database.py` defines all collections, their JSON Schema validators, and indexes in a single idempotent `setup()` function called at application startup. This ensures the database schema is automatically initialized on first run and remains immutable through subsequent executions.

---

## Functions

### `get_client()`

Builds the MongoDB connection URI from environment variables and returns a `MongoClient` instance. Authentication credentials are securely loaded from environment variables (never hardcoded or exposed in documentation).

### `setup()`

Creates all collections and indexes if they do not already exist. Uses `db.list_collection_names()` to check existence before creation — making the function fully **idempotent** (safe to run multiple times without duplicating data).

**Recommended load order:**
1. `domain_indicators` — reference table, must exist before others
2. `energy_losses_tariff` — annual regulatory data per distributor
3. `distribution_indices` — operational records, loaded in chunks
4. `consumer_units_pj` — geographic UC data (AT + MT)

## Collections

### 1. `domain_indicators`

**Source:** ANEEL energy domain indicators  
**Granularity:** One document per indicator  
**Validation:** `strict`

Reference table for energy indicators selected for the Tecsys project. The `_id` is the indicator acronym itself (natural key), enabling O(1) lookups with no extra index.

#### Schema

| Field | Type | Required | Description |
|---|---|---|---|
| `_id` | string | ✅ | Indicator acronym (e.g., `"DEC"`, `"FEC"`). Natural key. Mapped from `SigIndicador`. |
| `description` | string | ✅ | Full indicator description as published by ANEEL. Mapped from `DscIndicador`. |
| `group` | string (enum) | ✅ | Thematic group. See groups table below. Mapped from classification logic. |
| `generation_date` | string | ✅ | Dataset generation date `YYYY-MM-DD`. Mapped from `DatGeracaoConjuntoDados`. |

#### Thematic Groups (enum)

| Value | Indicators |
|---|---|
| `continuity` | DEC, FEC and variants, NumOcorr, Nie, Pnie, NDIACRI |
| `voltage_quality` | DRP, DRC, ICC, TNS, DICVLD, DMICVLD, FICVLD and variants |
| `dimensioning` | ERP, PNIT, AREA, NumCon, NUCTCO and variants |
| `compensations` | COMPCONF, COMPCONT, PGUC\*, PGUG\*, QTUC\*, QTUG\* |
| `market` | CMMT, CMMTCO, CMMTIN, CMMTRE, CMMTRU and variants |

#### Indexes
No additional indexes — `_id` is already indexed by MongoDB.

### 2. `distribution_indices`

**Source:** ANEEL operational data   
**Granularity:** One document per measurement (indicator + distributor + UC set + year + period)  
**Validation:** `strict`

Core operational collection. Stores quality and continuity measurements reported by distributors to ANEEL.

#### Schema

| Field | Type | Required | Description |
|---|---|---|---|
| `_id` | objectId | auto | Auto-generated by MongoDB. |
| `agent_acronym` | string | ✅ | Distributor acronym (e.g., `"ENEL SP"`). Mapped from `SigAgente`. |
| `cnpj_number` | string | ✅ | Distributor CNPJ. Mapped from `NumCNPJ`. |
| `consumer_unit_set_id` | string | ✅ | UC set identifier. Join key with `consumer_units_pj`. Mapped from `IdeConjUndConsumidoras`. |
| `consumer_unit_set_description` | string | ✅ | UC set description. Mapped from `DscConjUndConsumidoras`. |
| `indicator_type_code` | string | ✅ | Indicator acronym. References `domain_indicators._id`. Mapped from `SigIndicador`. |
| `year` | int | ✅ | Measurement year (2021–2025). Mapped from `AnoIndice`. |
| `period` | int (1–12) | ✅ | Measurement month. Mapped from `NumPeriodoIndice`. |
| `value` | double \| null | ✅ | Indicator value. Null if original data is invalid. Mapped from `VlrIndiceEnviado`. |
| `generation_date` | string | ✅ | Dataset generation date `YYYY-MM-DD`. Mapped from `DatGeracaoConjuntoDados`. |

#### Indexes

| Name | Fields | Notes |
|---|---|---|
| `idx_indicator_year_agent` | `(indicator_type_code, year, agent_acronym)` | Main composite index — covers most analytical queries |
| `idx_set_indicator_year` | `(consumer_unit_set_id, indicator_type_code, year)` | Join with `consumer_units_pj` |
| `idx_cnpj` | `(cnpj_number)` | Lookup by distributor CNPJ |
| `idx_unique_measurement` | `(agent_acronym, consumer_unit_set_id, indicator_type_code, year, period)` | **Unique** — ensures ETL idempotency via upsert |

**Validation:** `strict`

Stores regulatory energy loss data per distributor and tariff process. Covers losses in the basic network (AT), technical and non-technical losses (MT/AT) with their respective costs in BRL.

> **Note:** Files 1, 2 and 4 from the ANEEL losses portal were discarded:
> - File 1 — metrics exclusively over BT market
> - File 2 — subset of File 3, no additional information
> - File 4 — PNT percentages over BT market only

| Field | Type | Required | Description |
|---|---|---|---|
| `_id` | objectId | auto | Auto-generated by MongoDB. |
| `distributor` | string | ✅ | Distributor name. Mapped from `Distribuidora`. |
| `distributor_slug` | string | ✅ | Distributor slug for joining with other collections. Mapped from `sigla`. |
| `state` | string | ✅ | Full state name (e.g., `"Amazonas"`). Mapped from `Estado`. |
| `uf` | string | ✅ | State abbreviation (e.g., `"AM"`). Mapped from `UF`. |
| `process_date` | string | ✅ | Tariff process date `YYYY-MM-DD`. Mapped from `Data do Processo`. |
| `tme_brl_mwh` | double | ✅ | Average energy tariff in BRL/MWh. Mapped from `TME (R$/MWh)`. |
| `basic_network_loss_mwh` | double | ✅ | Basic network losses (AT) in MWh. Mapped from `Perdas na Rede Básica (MWh)`. |
| `technical_loss_mwh` | double | ✅ | Technical losses (MT/AT) in MWh. Mapped from `Perdas Técnicas (MWh)`. |
| `non_technical_loss_mwh` | double | ✅ | Non-technical losses — theft/fraud (MT/AT) in MWh. Mapped from `Perdas Não Técnicas (MWh)`. |
| `basic_network_loss_cost_brl` | double | ✅ | Cost of basic network losses in BRL. Mapped from `Custo Perdas na Rede Básica (R$)`. |
| `technical_loss_cost_brl` | double | ✅ | Cost of technical losses in BRL. Mapped from `Custo Perdas Técnicas (R$)`. |

| `parcel_b_brl` | double | ✅ | Parcel B regulatory cost in BRL. Mapped from `Parcela B (R$)`. |
| `required_revenue_brl` | double | ✅ | Required revenue in BRL. Mapped from `Receita Requerida (R$)`. |

### 4. `consumer_units_pj`

**Sources:**
- `UCAT_PJ.csv` — High Voltage Legal Entity Consumer Units (AT)
- `UCMT_PJ.csv` — Medium Voltage Legal Entity Consumer Units (MT)

**Granularity:** One document per consumer unit  
**Validation:** `strict`

Unified collection for AT and MT consumer units using a **flexible schema**. The `voltage_level` field (`"AT"` or `"MT"`) acts as the main discriminator, determining which demand and energy subdocument structure applies.

#### Voltage Level Discriminator

| `voltage_level` | Demand fields | Energy fields | AT-only fields | MT-only fields |
|---|---|---|---|---|
| `"AT"` | `demand.peak`, `demand.off_peak` | `energy.peak`, `energy.off_peak` | `at_transformer_code` | — |
| `"MT"` | `demand.total` | `energy.total` | — | `mt_segment_code`, `at_transformer_unit`, `no_network`, `base_date` |

> `voltage_level` is **not a column in the CSV files** — it is derived by the ETL based on which file is being processed (`UCAT_PJ.csv` → `"AT"`, `UCMT_PJ.csv` → `"MT"`).

#### Demand Subdocument

Flexible subdocument storing monthly demand measurements (kW). Structure depends on voltage level:

```json
// AT — Peak and off-peak demand measurements
"demand": { "peak": [12 values], "off_peak": [12 values] }

// MT — Total demand measurement
"demand": { "total": [12 values] }
```

| Field | Type | AT | MT | Description |
|---|---|---|---|---|
| `peak` | array (12) | ✅ | ❌ | [AT only] Peak demand — 12 monthly values (kW). Values can be `double`, `int`, or `null`. Mapped from `DEM_P_01` to `DEM_P_12`. |
| `off_peak` | array (12) | ✅ | ❌ | [AT only] Off-peak demand — 12 monthly values (kW). Values can be `double`, `int`, or `null`. Mapped from `DEM_F_01` to `DEM_F_12`. |
| `total` | array (12) | ❌ | ✅ | [MT only] Total demand — 12 monthly values (kW). Values can be `double`, `int`, or `null`. Mapped from `DEM_01` to `DEM_12`. |

#### Energy Subdocument

Flexible subdocument storing monthly energy measurements (kWh). Structure depends on voltage level:

```json
// AT — Peak and off-peak energy measurements
"energy": { "peak": [12 values], "off_peak": [12 values] }

// MT — Total energy measurement
"energy": { "total": [12 values] }
```

| Field | Type | AT | MT | Description |
|---|---|---|---|---|
| `peak` | array (12) | ✅ | ❌ | [AT only] Peak energy consumption — 12 monthly values (kWh). Values can be `double`, `int`, or `null`. Mapped from `ENE_P_01` to `ENE_P_12`. |
| `off_peak` | array (12) | ✅ | ❌ | [AT only] Off-peak energy consumption — 12 monthly values (kWh). Values can be `double`, `int`, or `null`. Mapped from `ENE_F_01` to `ENE_F_12`. |
| `total` | array (12) | ❌ | ✅ | [MT only] Total energy consumption — 12 monthly values (kWh). Values can be `double`, `int`, or `null`. Mapped from `ENE_01` to `ENE_12`. |

#### Indexes

| Name | Fields | Notes |

## Usage in FastAPI

