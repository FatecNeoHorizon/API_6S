# Non-Relational Database Documentation

## Overview

This MongoDB schema models physical infrastructure and operational metrics of Brazil's energy distribution networks. The design supports two primary objectives:

1. **Infrastructure Dimensioning:** Identify network capacity and geographic distribution patterns to assess market potential for Tecsys technology (sensing and telemetry).
2. **Loss Prediction:** Geographically locate energy losses and continuity failures to guide infrastructure improvements and predict where Tecsys deployment would have the highest impact.

The data-driven approach enables regulatory compliance (ANEEL reporting) while creating actionable insights for reducing non-technical losses (theft, fraud) and improving service continuity.

## Technology Stack

**Database:** `tecsys` (MongoDB 6.0)  
**Driver:** PyMongo  
**Connection:** Configured via environment variables using `python-dotenv`

## Setup Overview

`database.py` defines all collections, their JSON Schema validators, and indexes in a single idempotent `setup()` function called at application startup. This ensures the database schema is automatically initialized on first run and remains immutable through subsequent executions.

**Recommended load order:**
1. `domain_indicators` — reference table, must exist before others
2. `energy_losses_tariff` — annual regulatory data per distributor
3. `distribution_indices` — operational records, loaded in chunks
4. `consumer_units_pj` — geographic UC data (AT + MT)
5. `geodatabases` — GDB import control record (must exist before GDB layers)
6. `municipalities` — municipality polygons from GDB
7. `substations` — substation polygons from GDB
8. `distribution_transformers` — transformer points from GDB
9. `mt_network_segments` — MT network segments from GDB
10. `at_network_segments` — AT network segments from GDB

---

## Functions

### `get_client()`

Builds the MongoDB connection URI from environment variables and returns a `MongoClient` instance. Authentication credentials are securely loaded from environment variables (never hardcoded or exposed in documentation).

### `setup()`

Creates all collections and indexes if they do not already exist. Uses `db.list_collection_names()` to check existence before creation — making the function fully **idempotent** (safe to run multiple times without duplicating data).

## Collections

### 1. `domain_indicators`

**Source:** ANEEL energy domain indicators  
**Granularity:** One document per indicator  
**Validation:** `strict`

Reference table for energy indicators selected for the Tecsys project. The `_id` is the indicator acronym itself (natural key), enabling O(1) lookups with no extra index.

#### Schema

| Field | Type | Required | Description |
|---|---|---|---|
| `_id` | string | ✅ | Indicator acronym (e.g., `"DEC"`, `"FEC"`). Natural key. Mapped from `SigIndicador`. |
| `description` | string | ✅ | Full indicator description as published by ANEEL. Mapped from `DscIndicador`. |
| `group` | string (enum) | ✅ | Thematic group. See groups table below. |
| `generation_date` | string | ✅ | Dataset generation date `YYYY-MM-DD`. Mapped from `DatGeracaoConjuntoDados`. |

#### Thematic Groups (enum)

| Value | Indicators |
|---|---|
| `continuity` | DEC, FEC and variants, NumOcorr, Nie, Pnie, NDIACRI |
| `voltage_quality` | DRP, DRC, ICC, TNS, DICVLD, DMICVLD, FICVLD and variants |
| `dimensioning` | ERP, PNIT, AREA, NumCon, NUCTCO and variants |
| `compensations` | COMPCONF, COMPCONT, PGUC\*, PGUG\*, QTUC\*, QTUG\* |
| `market` | CMMT, CMMTCO, CMMTIN, CMMTRE, CMMTRU and variants |

#### Indexes
No additional indexes — `_id` is already indexed by MongoDB.

### 10. `at_network_segments`

**Source:** GDB layer `SEG_SDAT`  
**Geometry:** MultiLineString  
**Validation:** `moderate`

Physical segments of the High Voltage network. Nearly identical structure to `mt_network_segments` with two differences: no `transformer_unit` field, and `position` includes `"O"` (Other) in addition to `"D"` (Distribution).

> **Observed field values from real GDB data:**
> `FAS_CON: ['ABC']` | `POS: ['D', 'O']` | `MAT_FAS: ['4', '3', '0', '1']` | `GEOM_CAB: ['2', '0']`

#### Schema

Same as `mt_network_segments` except:
- No `transformer_unit` field
- `position` enum includes `"O"` (Other) in addition to `"D"`
- Neutral conductor ampacity mapped from `CM_NEU` (not `CP_CM_NEU`)

#### Indexes

Same as `mt_network_segments`.
